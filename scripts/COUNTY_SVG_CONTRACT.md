# County SVG Rendering Contract

**Purpose**: Define the exact schema and naming convention that generated county SVGs must follow to render correctly in the analytics dashboard.

## SVG File Structure

### Location & Naming
- **Path**: `calculators/wealth-tax-calculator/wordpress/wealth-tax-calculator/data/states/{stateSlug}/{stateSlug}-counties.svg`
- **Example**: `data/states/california/california-counties.svg` for California
- **State slug**: Two-letter state code in lowercase (ca, ny, tx, mi, etc.)

### SVG Root Element
```xml
<svg xmlns="http://www.w3.org/2000/svg" 
     viewBox="0 0 {width} {height}" 
     id="wtc-state-map-{STATE_CODE}"
     role="img"
     aria-label="{State} county map">
  <title>{State} Counties</title>
  <g class="wtc-counties">
    <!-- county path/rect elements -->
  </g>
</svg>
```

**Required attributes**:
- `xmlns="http://www.w3.org/2000/svg"` — SVG namespace
- `viewBox="0 0 W H"` — Bounding box; any reasonable bounds work as long as all county geometry fits
- `id="wtc-state-map-{STATE_CODE}"` — State code uppercase (MI, CA, TX, etc.)
- `role="img"` — Accessibility
- `aria-label="{State} county map"` — Accessible description
- `<title>{State} Counties</title>` — SVG title element

### County Elements (Path or Rect)
```xml
<path class="wtc-county"
      id="wtc-county-{slug}"
      data-county="{Display Name}"
      d="M..."/>

<!-- OR for rects (legacy, not recommended) -->
<rect class="wtc-county"
      id="wtc-county-{slug}"
      data-county="{Display Name}"
      x="0" y="0" width="100" height="100"/>
```

**Required attributes per county**:
- `class="wtc-county"` — Enables CSS styling and JS filtering
- `id="wtc-county-{slug}"` — Must be unique per state; used for SVG DOM queries
- `data-county="{Display Name}"` — Human-readable county name for tooltips
- For `<path>`: `d="M..."` with valid SVG path data
- For `<rect>`: `x`, `y`, `width`, `height` attributes

**No styling attributes required**: Class-based styling is applied by CSS in `admin-map.css`.

## County Slug Naming Convention

### Slug Generation Rules
Slugs must be generated by applying WordPress `sanitize_title()` equivalent, with state-specific exceptions.

**Default normalization** (`normalize_generic_county_slug` in PHP):
```
Input: "Los Angeles County"
1. Remove trailing descriptors: " county|parish|borough|municipality|census-area|city-and-borough" (case-insensitive)
   → "Los Angeles"
2. Apply sanitize_title() equivalent:
   - Convert to lowercase
   - Replace spaces with hyphens
   - Remove special characters (periods, apostrophes, etc.)
   - Result: "los-angeles"
3. Final ID: "wtc-county-los-angeles"
4. Final bucket (in analytics): "ca_county_los-angeles"
```

### Specific Examples by State
| State | County Name | After Step 1 | After Step 2 | Final Slug |
|-------|-----------|-------------|-------------|-----------|
| CA | Los Angeles County | Los Angeles | los-angeles | los-angeles |
| CA | San Francisco County | San Francisco | san-francisco | san-francisco |
| TX | De Witt County | De Witt | de-witt | de-witt |
| MI | St. Clair County | St. Clair | st-clair | st-clair |
| IL | DuPage County | DuPage | dupage | dupage |
| GA | DeKalb County | DeKalb | dekalb | dekalb |
| NY | New York County | New York | new-york | new-york |
| LA | St. Bernard Parish | St. Bernard | st-bernard | st-bernard |

### Exception: Michigan (Validated Against Hardcoded Lookup)
Michigan uses `normalize_county_slug()` instead, which:
1. Removes trailing " county" (case-insensitive)
2. Applies `sanitize_title()`
3. **Validates against hardcoded 83-county lookup table** [wealth-tax-calculator.php:1758]
4. Returns empty string if not found

This means Michigan slugs are pre-validated; other states are not.

## Bucket Format in Analytics Data

When a user in a county submits data, the analytics system generates a bucket key:
```
{stateCode}_county_{slug}
```

**Examples**:
- `ca_county_los-angeles` — Los Angeles County, California
- `tx_county_harris` — Harris County, Texas
- `mi_county_st-clair` — St. Clair County, Michigan
- `ny_county_new-york` — New York County, New York

**State code**: Always lowercase, two-letter US state code (ca, ny, mi, etc.)
**Slug part**: Must match exactly what `normalize_generic_county_slug()` (or Michigan's validation) produces from GeoIP county name data.

## Frontend Rendering Contract

### JavaScript SVG Template Loading
[admin-map.js:383-414](../calculators/wealth-tax-calculator/wordpress/wealth-tax-calculator/js/admin-map.js)

```javascript
// 1. Query template by state code
var templateRoot = document.getElementById('wtc-state-svg-template-' + stateCode);
// Expected format: <div id="wtc-state-svg-template-{STATECODE}" hidden>...SVG...</div>

// 2. Clone SVG
var svgTemplate = templateRoot.querySelector('svg');
var svgEl = svgTemplate.cloneNode(true);

// 3. Append to page
hostEl.appendChild(svgEl);

// 4. Query counties
var slug = extractCountySlug(county.bucket, stateCode);
// e.g., bucket="ca_county_los-angeles" → slug="los-angeles"
var path = svgEl.querySelector('#wtc-county-' + slug);
// Query: #wtc-county-los-angeles

// 5. Get bounding box and render bubble
var bbox = path.getBBox();  // Returns {x, y, width, height}
// Circle center: (bbox.x + bbox.width/2, bbox.y + bbox.height/2)
```

### Requirements for Rendering
1. **SVG elements with IDs**: Every county in analytics data must have a matching `#wtc-county-{slug}` element in the SVG.
2. **Valid geometries**: Both `<path>` and `<rect>` elements work via `getBBox()`.
3. **Unique IDs**: No duplicate `id="wtc-county-..."` values within a single state SVG.
4. **Bounding box consistency**: Small, irregular paths still work; `getBBox()` computes the smallest rectangle that contains them.

### Silent Failure Behavior
If `querySelector('#wtc-county-{slug}')` returns `null` (element doesn't exist), the JavaScript function returns without rendering anything:
```javascript
if (!path || typeof path.getBBox !== 'function') {
    return;  // No bubble for this county — silently skips
}
```

This means **mismatched slugs are invisible errors** during rendering but the bucket data is still recorded. No error message appears in the browser console.

## Validation Checklist for Generated SVGs

Before deployment, each generated SVG must pass:

- ✅ **File exists**: `data/states/{stateSlug}/{stateSlug}-counties.svg`
- ✅ **Valid XML**: Parses without errors in browser
- ✅ **Root element**: Has `xmlns`, `viewBox`, `id="wtc-state-map-{CODE}"`
- ✅ **County container**: Has `<g class="wtc-counties">`
- ✅ **No duplicate IDs**: No two `id="wtc-county-..."` values are identical within the file
- ✅ **Correct slug format**: All IDs match pattern `[a-z0-9\-]+` (lowercase, numbers, hyphens only)
- ✅ **County count**: Number of `<path class="wtc-county">` or `<rect class="wtc-county">` elements matches expected county count for state
- ✅ **Slug parity**: Every expected slug from the canonical manifest has a corresponding element
- ✅ **Geometry validity**: All `<path>` elements have valid `d` attributes; all `<rect>` elements have valid `x`, `y`, `width`, `height`
- ✅ **getBBox compatibility**: Elements can be queried via `getBBox()` (automatic for valid SVG elements)

## Backend Bucket Generation Contract

### Where County Buckets Come From
[wealth-tax-calculator.php:3969 — `get_geo_context()`]

1. **IP Geolocation**: ipapi.co API returns county name (or one of: country_code, region_code, city, county, county_name, state_district, district)
2. **Normalization**: Pass through `normalize_generic_county_slug()` or Michigan's `normalize_county_slug()`
3. **Bucket key**: Format as `{stateCode}_county_{slug}`

### Slot in Analytics Option Storage
[wealth-tax-calculator.php:1727 — `build_summary()`]

Buckets collected as keys in `county_counts` array:
```php
$summary = [
    'county_counts' => [
        'ca_county_los-angeles' => 125,  // 125 submissions from LA County
        'ca_county_san-diego' => 98,
        'tx_county_harris' => 203,
        ...
    ],
];
```

State code and slug case: **always lowercase** in final bucket keys.

## Exceptions and Known Issues

### Current Exceptions
1. **Michigan**: Uses hardcoded 83-county lookup; other states do not.
2. **California**: Currently uses rect tile placeholders; will be replaced with path geometry.

### Potential Slug Variants (Test Cases)
If ipapi.co returns these county names, normalize_generic_county_slug() will produce:

| GeoIP Return | Generated Slug | Risk |
|--------------|----------------|------|
| "DeKalb County" | `dekalb` | If SVG created with `de-kalb`, **MISMATCH** |
| "De Kalb County" | `de-kalb` | If `dekalb` in SVG, **MISMATCH** |
| "St. Louis County" | `st-louis` | Period becomes hyphen — usually safe |
| "Saint Louis County" | `saint-louis` | If SVG uses `st-louis`, **MISMATCH** |
| "DuPage County" | `dupage` | Safe |
| "Du Page County" | `du-page` | Unlikely but possible variant |

**Mitigation**: During generation, validate that manifest expected slugs match what `normalize_generic_county_slug()` actually produces for that state's canonical county names.
